# План создания макетов LMS (по материалам Image #1-#10)

## 1. Цель
Собрать целостный набор UX/UI-макетов для блока "Курсы" в LMS: от списка курсов и страницы курса до детальной страницы задания, включая состояния выполнения, оценивания, вложений, модулей, мобильную версию и сопутствующие блоки (оценки, календарь, диаграмма Ганта).

## 2. Что извлечено из исходных материалов
- `Image #3`: экран списка курсов (`Все разделы -> Мои курсы`) с поиском и карточкой-таблицей курсов.
- `Image #1`, `Image #5`, `Image #6`: desktop-варианты страницы конкретного курса с разными левыми панелями (полная, сокращенная, иконки).
- `Image #8`: расширенная структура курса с модулями/темами, разными типами карточек (инфо, видео, задание), блоком диаграммы Ганта.
- `Image #2`: раскрытая карточка задания (внутри списка курса) со списком файлов и комментарием.
- `Image #4`, `Image #7`: детальная страница задания (full page) со статусами, оценкой, файлами, комментариями, действиями.
- `Image #9`: мобильная версия списка модулей/задач (вертикальный стек карточек).
- `Image #10`: приоритеты работ по фронту (задачи, блоки задач, оценки, страница задачи, боковая плашка, календарь, диаграмма Ганта, темы).

## 3. Границы проекта макетов
### В scope (обязательно)
- Экран "Мои курсы" (список курсов).
- Экран "Курс" (левая навигация + контент модулей/задач).
- Карточки контента в курсе (информация / видео / задание / лабораторная / тест).
- Раскрытие/сворачивание карточек и модулей.
- Детальная страница задания.
- Состояния статусов выполнения и оценивания.
- Мобильная адаптация основных пользовательских сценариев.

### Дополнительно (после core)
- Блок календаря.
- Блок диаграммы Ганта.
- Полировка темы/визуальных тем (если требуется отдельный dark/light или вариативность).

## 4. Информационная архитектура (IA)
1. `Все разделы`
2. `Мои курсы`

6. `Карточка задания (collapsed/expanded)`
7. `Страница задания`
8. `Редактирование/удаление ответа` (как состояния/модалки)

## 5. Перечень макетов, которые нужно сделать
### 5.1 Список курсов (`Image #3`)
- Header с крупной строкой поиска.
- Контейнер "Мои курсы" + таблица/список курсов.
- Состояния строки курса: default / hover / active.
- Счетчик или индикатор справа.

### 5.2 Страница курса desktop (`Image #1`, `#5`, `#6`, `#8`)
- Вариант A: полная боковая колонка (аватар, поиск, разделы).
- Вариант B: боковая колонка-список сущностей курса.
- Вариант C: иконографическая узкая колонка.
- Основной контент: шапка курса, информационные блоки, модули, карточки задач.
- Поддержка длинных названий (перенос строк, обрезка, tooltip-логика).

### 5.3 Карточки в курсе (`Image #2`, `#8`, `#9`)
- Типы карточек: `Инфо`, `Видео`, `Задание`, `Лабораторная`, `Тест`.
- Состояния: collapsed / expanded / completed / with-grade / with-comment / with-files.
- Элементы: иконки типа, чекбокс выполнения, стрелка раскрытия, метаданные (дата, автор, период).
- Вложенный список файлов: строки с иконкой документа, разделителями, правым чекбоксом.

### 5.4 Детальная страница задания (`Image #4`, `#7`)
- Левый инфо-столбец: статус выполнения, состояние оценивания, дата/время, файлы, комментарий.
- Основной контент: название задания, текст задания/бриф, CTA-кнопки (`Редактировать`, `Удалить`).
- Состояния:
  - выполнено/не выполнено,
  - оценено/не оценено,
  - с файлом/без файла,
  - с комментарием/без комментария.

### 5.5 Мобильная версия (`Image #9`)
- Вертикальный поток модулей и карточек.
- Тап-зоны для чекбоксов/стрелок не меньше 40x40.
- Сохранение иерархии: модуль -> карточки -> раскрытие.
- Проверка читаемости длинных строк и плотности контента.

## 6. Компонентная система (что вынести в reusable)
- `TopSearchBar`
- `Sidebar (3 variants)`
- `CourseHeader`
- `ModuleHeader`
- `ContentCard` (параметр type)
- `TaskCardExpanded`
- `StatusBadge` (выполнение/оценка)
- `FileRow`
- `MetaInfoRow` (автор/дата/период)
- `ActionButtonsGroup`
- `LegendBlock` (цветовые статусы)

## 7. Матрица состояний (обязательная к отрисовке)
- Выполнение: `Не начато` / `В процессе` / `Выполнено` / `Просрочено`.
- Оценивание: `Не оценено` / `Удовлетворительно` / `Хорошо` / `Отлично` / `Неудовл.`.
- Контент: `без вложений` / `1+ вложений`.
- Комментарии: `нет` / `есть комментарий преподавателя` / `есть комментарий студента`.
- Раскрытие: `свернуто` / `раскрыто`.

## 8. Базовые UI-правила для макетов
- Шаг сетки: 8 px (внутренние отступы кратны 8).
- Радиусы: большие контейнеры 14-20 px; инпуты/кнопки 10-14 px.
- Типографика: единая шкала (например 12/14/16/20/36).
- Контраст текста: body-текст не ниже WCAG AA для основного контента.
- Для длинных заголовков: max 2 строки + мягкое усечение.

## 9. Поэтапный план производства макетов
### Этап 0. Подготовка
- Зафиксировать флоу пользователя (от списка курсов к задаче).
- Утвердить список экранов и breakpoints.
- Собрать примитивы (цвета, шрифты, тени, бордеры).

### Этап 1. Каркас и навигация
- Макеты: список курсов + базовая страница курса.
- Отрисовать 3 варианта боковой панели.
- Проверить компоновку на длинном контенте.

### Этап 2. Блоки задач в курсе (приоритет из `Image #10`)
- Построить компонент карточки задания.
- Добавить expanded-состояние с файлами и комментарием.
- Отрисовать типы карточек (инфо/видео/тест/лабораторная/задание).

### Этап 3. Оценки и статусные блоки
- Сделать единую систему статус-бейджей и цветовых состояний.
- Отрисовать легенду статусов и применить к карточкам/детальной странице.

### Этап 4. Страница задания
- Собрать desktop-макет страницы задания.
- Добавить все ключевые состояния (с файлом/без, оценено/не оценено).
- Проверить кнопочные сценарии (`Редактировать`, `Удалить`).

### Этап 5. Модули/темы
- Макеты структуры: модуль -> тема -> карточки.
- Состояния раскрытия модулей и вложенных элементов.

### Этап 6. Календарь и диаграмма Ганта
- Встроить блок календаря в экосистему курса.
- Встроить блок диаграммы Ганта (как в `Image #8`) в общем стиле.

### Этап 7. Mobile + QA
- Полный mobile-проход по ключевым экранам.
- Проверка интерактивных зон, читабельности и отступов.
- Подготовка handoff: компоненты, спецификации, состояния.

## 11. Definition of Done для дизайна
- Для каждого экрана есть desktop + mobile версия.
- Для каждой карточки есть минимум `collapsed` и `expanded`.
- Все статусные состояния покрыты отдельными фреймами.
- Поведение длинного текста и пустых состояний отрисовано.
- Компоненты собраны в библиотеку и переиспользуются.
- Передача в разработку содержит размеры, отступы, цвета, типографику, интерактивные состояния.

## 12. Риски и что уточнить до старта
- Единый источник правды по статусам оценивания (финальные названия и цвета).

- Нужна ли отдельная dark theme или достаточно текущей светлой.
- Точные правила сортировки/группировки для модулей и тем.
- Какие блоки календаря/Ганта интерактивные, а какие только обзорные.

## 16. Готовые промпты для 18 агентов (с учетом текущей архитектуры проекта)
### 16.1 Архитектурный префикс (добавлять в каждый промпт)
- Стек: `Flask + Jinja2 + статические CSS/JS`, без React/Vue и без смены технологического стека.
- Backend-вход: `main.py` + blueprints `routes/my_curse.py`, `routes/gantt.py`, `routes/input_file.py`.
- Шаблоны: `templates/*`; стили и скрипты: `static/*`.
- Визуальные референсы без доступа к изображениям брать из текстовой спецификации в разделе `21`.
- Ключевые экраны уже разведены по файлам:
  - список курсов: `templates/courses/courses.html`, `static/courses/courses.css`
  - страница курса: `templates/my_curse/my_curse.html`, `static/my_curse/my_curse.css`, `static/my_curse/my_curse_block.css`, `static/my_curse/scripts/script.js`
  - Гант: `templates/gantt/gantt_dhtmlx.html`, `routes/gantt.py`, `static/gantt/*`
  - файловый блок: `templates/input_file/input.html`, `static/input_file/input_file.css`, `static/input_file/input_file.js`
  - базовые блоки: `templates/Base/Base.html`, `templates/BlockFiles/BlockFiles.html`, `templates/BlockVideo/BlockVideo.html`
- Нейминг и существующие пути не ломать: сохранять текущие имена (`my_curse`, `courses`, `gantt`, `input_file`).
- Любая задача агента должна завершаться списком `какие файлы затронуты` и `какие контракты не тронуты`.
